# 目录约束（v1）

## 1. 根目录约束
- `core/`：配置、异常、通用模型。
- `router/`：路由聚合、依赖注入、HTTP边界。
- `modules/`：业务域实现（禁止散落 *_service 新目录）。
- `store/`：数据库模型与仓储。
- `runtime/`：运行时生成数据目录（如图表导出临时文件）。
- `templates/`、`static/`：兼容层；`/analysis` 不再依赖模板入口。
- `frontend/`：Vite + Vue3 + TS 前端工程。
- `tests/`：按 `api/domain/integration/e2e` 分层。

## 2. modules 二阶约束
- 业务域目录仅允许：`admin`、`isochrone`、`poi`、`h3`、`road`、`export`、`providers`。
- Provider 统一放到 `modules/providers/*`（当前为 `amap`）。
- 已下线 legacy 目录：`grid_h3`、`road_syntax`、`export_bundle`、`gaode_service`、`isochrone_service`、`generated_charts`。

## 3. 测试分层约束
- `tests/api/`：HTTP接口级测试。
- `tests/domain/`：纯业务核心测试。
- `tests/integration/`：跨模块协同测试。
- `tests/e2e/`：端到端行为测试。

## 4. 仓库卫生约束
- 禁止提交：`__pycache__/`、`*.pyc`、`.pytest_cache/`、`.playwright-cli/`、`*.bak`。
- 禁止引用 legacy import：`modules.grid_h3`、`modules.road_syntax`、`modules.export_bundle`、`modules.gaode_service`。
- 禁止恢复旧 analysis 链路：`templates/analysis.html`、`static/js/analysis/**`。
- 使用 `scripts/check_repo_hygiene.sh` 做结构与垃圾文件检查。
